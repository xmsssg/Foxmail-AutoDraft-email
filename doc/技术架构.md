# 发货单邮件草稿自动生成工具技术架构

## 1. 架构定位

程序安装在业务人员电脑上，采用本地单机架构，不依赖服务器，不连接邮箱服务器协议。

```text
业务人员电脑
│
├─ AutoEmail 本地程序
│  ├─ 文件夹监听
│  ├─ 文件稳定检测
│  ├─ 防重复处理
│  ├─ 邮件草稿构造
│  ├─ Foxmail MAPI XML 导入
│  ├─ 本地数据库
│  └─ 本地日志
│
├─ Foxmail
│  └─ 草稿箱
│
└─ 客户发货单文件夹
   └─ *.xlsx
```

## 2. 推荐技术栈

```text
语言：Python
界面：托盘程序，后续可扩展 PySide6 / Tkinter 设置界面
打包：PyInstaller
数据库：SQLite
配置：JSON
日志：logging + 滚动日志
文件监听：watchdog
Windows 自动化：pywinauto 或 pywin32
```

选择 Python 的原因：

```text
1. 文件监听、Hash、XML、SQLite 支持成熟
2. Windows 自动化生态较完整
3. 开发速度快，适合本地工具
4. 可用 PyInstaller 打包为 exe，业务电脑无需安装 Python
```

## 3. 数据库选型

推荐使用：

```text
SQLite
```

数据库文件建议位置：

```text
D:\Study\AutoEmail\data\draft_records.sqlite
```

选择 SQLite 的原因：

```text
1. 单机使用，不需要数据库服务器
2. 跟随程序安装，维护成本低
3. 适合记录处理历史、Hash、防重复、失败日志
4. 支持事务，避免程序中断导致状态混乱
5. 可直接备份一个 .sqlite 文件
6. Python 原生支持 sqlite3
```

不推荐：

```text
MySQL / PostgreSQL：对业务电脑过重，需要服务和维护
Access：兼容性和自动化维护不如 SQLite
纯 JSON：防重复、查询、状态更新、并发写入不稳
Excel 做数据库：不适合长期记录处理状态
```

如果部署到多台业务电脑：

```text
每台电脑维护自己的本地 SQLite
不共享数据库
不做中心化同步
```

## 4. 数据表设计

```sql
CREATE TABLE IF NOT EXISTS draft_records (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    customer_id TEXT NOT NULL,
    customer_name TEXT NOT NULL,

    file_path TEXT NOT NULL,
    file_name TEXT NOT NULL,
    file_size INTEGER NOT NULL,
    file_mtime TEXT NOT NULL,
    file_hash TEXT NOT NULL,

    subject TEXT,
    to_recipients TEXT,
    cc_recipients TEXT,
    body TEXT,

    mapi_xml_path TEXT,
    eml_path TEXT,

    import_status TEXT NOT NULL,
    import_message TEXT,
    imported_at TEXT,

    foxmail_msg_id INTEGER,
    foxmail_mail_path TEXT,

    status TEXT NOT NULL,
    error_message TEXT,

    created_at TEXT NOT NULL,
    updated_at TEXT NOT NULL,

    UNIQUE(file_path, file_hash)
);
```

建议增加索引：

```sql
CREATE INDEX IF NOT EXISTS idx_draft_records_status
ON draft_records(status);

CREATE INDEX IF NOT EXISTS idx_draft_records_customer_id
ON draft_records(customer_id);

CREATE INDEX IF NOT EXISTS idx_draft_records_created_at
ON draft_records(created_at);
```

## 6. 核心模块职责

### scanner.py

```text
扫描客户目录
发现新增 Excel
过滤 ~$ 临时文件
把候选文件交给后续流程
```

### file_stability.py

```text
检查文件大小是否稳定
检查 mtime 是否稳定
确认 Excel 已保存完成
```

### duplicate_guard.py

```text
计算文件 SHA256
根据 file_path + file_hash 判断是否已处理
防止重复生成和重复导入
```

### eml_builder.py

```text
生成 .eml 备查文件
不作为正式导入通道
用于审计、排查、留档
```

### mapi_xml_builder.py

```text
复制 Excel 附件到 %TEMP%\FMMAPI
生成 %TEMP%\FMMapi.xml
写入 Subject / Receivers / Attachments / Content
```

### foxmail_mapi_importer.py

```text
调用 Foxmail.exe MAPI: FMMapi.xml
等待 Foxmail 写信窗口出现
激活窗口
执行 Ctrl+S 保存草稿
验证草稿新增
返回导入结果
```

### draft_generator.py

```text
协调完整生成流程
更新数据库状态
处理失败和重试
```

### repository.py

```text
初始化 SQLite 表结构
新增处理记录
更新状态
查询重复记录
查询失败记录
```

## 8. 主流程

```text
发现 Excel
  ↓
过滤 ~$ 临时文件
  ↓
等待文件大小和 mtime 稳定
  ↓
计算 SHA256
  ↓
查询 SQLite 是否已 imported
  ↓
生成 .eml 备查文件
  ↓
复制附件到 %TEMP%\FMMAPI
  ↓
生成 FMMapi.xml
  ↓
调用 Foxmail.exe MAPI: FMMapi.xml
  ↓
等待 Foxmail 写信窗口
  ↓
Ctrl+S 保存草稿
  ↓
验证 draft.box / mId_bId.map 新增草稿
  ↓
写入 status=imported
```
