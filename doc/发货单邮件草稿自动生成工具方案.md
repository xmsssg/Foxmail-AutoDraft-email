# 发货单邮件草稿自动生成工具方案

## 1. 最终定位

工具名称：

```text
发货单邮件草稿自动生成工具
```

工具职责：

```text
监听客户文件夹中新 Excel 发货单
生成邮件草稿描述文件
调用 Foxmail MAPI 机制生成带附件草稿
保存到 Foxmail 草稿箱
记录处理状态
```

工具不做：

```text
不发送邮件
不连接 SMTP / POP / IMAP
不连接网易云企业邮箱服务器
不读取邮箱内容
不删除邮件
不解析 Excel 内容
不做发货业务管理
```

## 2. 业务流程

```text
业务人员制作 Excel 发货单
        ↓
保存到客户固定文件夹
        ↓
工具发现新 Excel
        ↓
等待文件保存完成
        ↓
计算 Hash 并防重复
        ↓
根据客户配置生成邮件主题、收件人、正文
        ↓
复制 Excel 到 Foxmail MAPI 临时附件目录
        ↓
生成 FMMapi.xml
        ↓
调用 Foxmail.exe MAPI: FMMapi.xml
        ↓
Foxmail 打开带附件写信窗口
        ↓
工具自动 Ctrl+S 保存为草稿
        ↓
业务人员在 Foxmail 草稿箱检查
        ↓
业务人员手动发送
```

工具结束点：

```text
草稿已导入 Foxmail
```

不是：

```text
邮件已发送
```

## 3. 已验证导入方式

正式方案不使用普通 `.eml` 直接导入。实测稳定方案是 Foxmail 的 MAPI XML 机制。

核心路径：

```text
%TEMP%\FMMapi.xml
%TEMP%\FMMAPI\
```

调用方式：

```text
D:\Program Files\Foxmail\Foxmail.exe MAPI: %TEMP%\FMMapi.xml
```

调用后对 Foxmail 写信窗口执行：

```text
Ctrl+S
```

实测成功结果：

```text
附件：D:\xusm\桌面\苏州方林送货单--20260525.xlsx
生成草稿编号：176
草稿文件：D:\Program Files\Foxmail\Storage\xusm@fanglin.cn\Mails\16\5\176
草稿大小：21501 字节
draft.box 包含 176
mId_bId.map 中 176 -> BoxId=4
重启 Foxmail 后仍存在
```

## 4. FMMapi.xml 格式

示例：

```xml
<FMMAPI version="1.0">
  <Subject>
    <![CDATA[苏州方林送货单--20260525]]>
  </Subject>
  <Receivers>
    <Receiver email="xusm@fanglin.cn" type="1"/>
  </Receivers>
  <Attachments>
    <attachment
      name="苏州方林送货单--20260525.xlsx"
      path="C:\Users\FL010950\AppData\Local\Temp\FMMAPI\苏州方林送货单--20260525.xlsx"/>
  </Attachments>
  <Content>
    <![CDATA[您好，附件为苏州方林送货单--20260525，请查收。]]>
  </Content>
</FMMAPI>
```

建议直接写邮箱地址：

```xml
<Receiver email="xusm@fanglin.cn" type="1"/>
```

抄送：

```xml
<Receiver email="cc@example.com" type="2"/>
```

## 5. 邮件生成规则

以文件：

```text
苏州方林送货单--20260525.xlsx
```

生成：

```text
主题：苏州方林送货单--20260525
附件名：苏州方林送货单--20260525.xlsx
正文：客户配置中的固定正文
收件人：客户配置
抄送人：客户配置，可选
```

临时附件复制到：

```text
%TEMP%\FMMAPI\苏州方林送货单--20260525.xlsx
```

## 6. 状态设计

```text
pending
generated
importing
imported
failed
ignored
```

含义：

| 状态 | 说明 |
| --- | --- |
| pending | 已发现，等待处理 |
| generated | 邮件草稿描述文件已生成 |
| importing | 正在调用 Foxmail |
| imported | 已成功保存到 Foxmail 草稿箱 |
| failed | 生成或导入失败 |
| ignored | 被规则忽略，例如 ~$ 临时文件 |

最终成功状态：

```text
status = imported
import_status = success
```

## 9. Foxmail 导入模块

建议类名：

```python
class FoxmailMapiImportResult:
    success: bool
    message: str
    imported_at: str | None
    foxmail_msg_id: int | None
    foxmail_mail_path: str | None


class FoxmailMapiDraftImporter:
    def import_draft(
        self,
        subject: str,
        body: str,
        to_recipients: list[str],
        cc_recipients: list[str],
        attachment_path: str,
    ) -> FoxmailMapiImportResult:
        ...
```

核心职责：

```text
1. 创建 %TEMP%\FMMAPI\
2. 复制 Excel 附件到 %TEMP%\FMMAPI\
3. 生成 %TEMP%\FMMapi.xml
4. 调用 Foxmail.exe MAPI: FMMapi.xml
5. 等待 Foxmail 写信窗口出现
6. 激活窗口
7. 执行 Ctrl+S
8. 检查 Foxmail 本地草稿是否新增
9. 返回导入结果
```

## 12. 防重复规则

如果：

```text
file_path + file_hash 已存在
且 status = imported
```

则：

```text
不重复生成
不重复导入
```

如果：

```text
file_path 相同
file_hash 不同
```

则认为是新版本，允许重新生成和导入。

输出文件名追加时间戳：

```text
苏州方林送货单--20260525_20260602_103012.eml
```

## 13. 异常处理

导入失败时：

```text
保留 Excel 原文件
保留 .eml
保留 FMMapi.xml
记录 failed
记录 error_message
下一轮不自动反复弹窗
```

除非：

```text
用户手动重试
或配置允许 retry_failed = true
```


## 15. 最终结论

最终方案定义为：

```text
一个本地自动化草稿工具。
它监听客户 Excel 发货单目录，生成邮件草稿描述文件，
通过 Foxmail MAPI XML 机制创建带附件草稿，
并自动保存到 Foxmail 草稿箱。
业务人员只负责在 Foxmail 中检查并手动发送。
```

最关键边界：

```text
自动生成并保存 Foxmail 草稿：是工具职责
发送邮件：不是工具职责
连接邮箱服务器协议：不是工具职责
```
