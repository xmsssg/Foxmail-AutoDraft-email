# 发货单邮件草稿自动生成工具任务清单

## 阶段 1：项目骨架

1. 创建项目目录结构：`app/core`、`app/config`、`app/storage`、`logs`、`output`、`tests`。
2. 创建 `settings.json`、`customers.json` 示例配置。
3. 初始化日志模块，支持滚动日志。
4. 初始化 SQLite 数据库和 `draft_records` 表。
5. 编写基础启动入口 `main.py`。

## 阶段 2：文件扫描

1. 实现客户配置读取。
2. 实现客户目录扫描。
3. 过滤 `~$` Excel 临时文件。
4. 只识别 `.xlsx`、`.xls`。
5. 实现文件稳定检测：文件大小和修改时间连续稳定。
6. 实现手动扫描入口。

## 阶段 3：防重复

1. 实现 SHA256 文件 Hash。
2. 查询 `file_path + file_hash` 是否已处理。
3. 已 `imported` 的文件跳过。
4. 同路径但 Hash 变化时允许重新处理。
5. 写入 `pending`、`ignored`、`failed` 状态。

## 阶段 4：邮件草稿构造

1. 根据 Excel 文件名生成主题。
2. 根据客户配置生成收件人、抄送人、正文。
3. 生成 `.eml` 备查文件。
4. 复制附件到 `%TEMP%\FMMAPI`。
5. 生成 `%TEMP%\FMMapi.xml`。
6. 保留生成产物路径到数据库。

## 阶段 5：Foxmail 导入

1. 实现 `foxmail_mapi_importer.py`。
2. 调用 `Foxmail.exe MAPI: FMMapi.xml`。
3. 等待 Foxmail 写信窗口出现。
4. 激活写信窗口。
5. 执行 `Ctrl+S` 保存草稿。
6. 等待 Foxmail 写入本地邮件库。
7. 验证新增草稿编号和 `BoxId=4`。
8. 成功写入 `status=imported`。
9. 失败写入 `status=failed` 和错误原因。

## 阶段 6：托盘程序

1. 实现托盘常驻。
2. 托盘菜单：立即扫描、重试失败、打开日志、打开输出目录、退出。
3. 成功导入后显示轻提示。
4. 失败时显示提示但不反复弹窗。
5. 支持开机启动配置。

## 阶段 7：异常与重试

1. Foxmail 路径不存在时提示配置错误。
2. Foxmail 窗口未出现时记录失败。
3. 文件被占用时等待或失败。
4. 导入失败保留 `.eml`、`FMMapi.xml`、日志。
5. 实现手动重试失败记录。
6. 防止同一失败任务无限自动重试。

## 阶段 8：测试验收

1. 新 Excel 出现后自动生成草稿。
2. 草稿主题等于 Excel 文件名不带扩展名。
3. 收件人、抄送人、正文正确。
4. Excel 作为附件存在于 Foxmail 草稿。
5. 业务人员可在 Foxmail 手动发送。
6. 同文件重复扫描不重复导入。
7. 同路径文件修改后可生成新草稿。
8. Foxmail 重启后草稿仍存在。
9. 程序不发送邮件、不连接邮箱服务器。

## 阶段 9：打包部署

1. 使用 PyInstaller 打包为 `AutoEmail.exe`。
2. 准备默认配置文件。
3. 准备部署目录：`config`、`data`、`logs`、`output`。
4. 编写安装/使用说明。
5. 在业务人员电脑上试运行。
6. 固化 Foxmail 路径和账号 Storage 路径配置。
