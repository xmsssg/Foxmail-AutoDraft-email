# 发货单邮件草稿自动生成工具发布升级说明

## 1. 新发布包结构

新版发布包采用“程序目录”和“业务数据目录”分离的结构。

推荐安装目录：

```text
D:\AutoEmail
├─ app
│  ├─ 发货单邮件草稿自动生成工具.exe
│  └─ _internal
├─ config
│  ├─ customers.json
│  └─ settings.json
├─ data
│  └─ draft_records.sqlite
├─ logs
│  └─ app.log
├─ output
│  ├─ generated_eml
│  └─ generated_mapi_xml
├─ doc
└─ 启动发货单草稿工具.vbs
```

## 2. 各目录职责

| 目录 | 说明 | 升级时是否覆盖 |
| --- | --- | --- |
| `app` | 程序本体 | 可以覆盖 |
| `config` | Foxmail 配置、客户配置 | 不覆盖 |
| `data` | 已处理记录数据库 | 不覆盖 |
| `logs` | 程序日志 | 不覆盖 |
| `output` | `.eml`、XML 备查文件 | 不覆盖 |
| `doc` | 使用说明和技术文档 | 可以覆盖 |

## 3. 首次安装流程

1. 将发布包解压到目标目录，例如：

```text
D:\AutoEmail
```

2. 双击：

```text
启动发货单草稿工具.vbs
```

3. 程序首次启动时，会自动创建：

```text
config
data
logs
output
```

4. 在程序界面中配置：

```text
Foxmail 程序路径
邮箱本地目录
客户名称
客户 Excel 文件夹
收件人
抄送人
正文
是否扫描子文件夹
是否开机自启
```

## 4. 后续版本升级流程

升级时不要覆盖整个安装目录。

正确操作：

1. 退出旧版程序。
2. 备份：

```text
D:\AutoEmail\config
D:\AutoEmail\data
```

3. 用新版发布包中的 `app` 目录覆盖旧的：

```text
D:\AutoEmail\app
```

4. 如文档有更新，可覆盖：

```text
D:\AutoEmail\doc
```

5. 不要覆盖：

```text
D:\AutoEmail\config
D:\AutoEmail\data
D:\AutoEmail\logs
D:\AutoEmail\output
```

6. 重新启动程序。

## 5. 为什么不能整包覆盖

因为业务电脑上的客户配置、邮箱配置、处理记录都保存在：

```text
config
data
```

如果整包覆盖这些目录，之前配置好的客户和处理记录会丢失。

## 6. 推荐升级前备份目录

每次升级前建议创建：

```text
D:\AutoEmail\backup\20260603_升级前备份
```

至少复制：

```text
D:\AutoEmail\config
D:\AutoEmail\data
```

## 7. 发布包打包原则

新版发布包不包含真实业务数据目录。

发布包只包含：

```text
app
doc
启动发货单草稿工具.vbs
```

程序首次启动时会自动创建运行目录。这样可以避免升级包覆盖业务电脑已有配置。

## 8. 一句话升级原则

```text
升级只替换 app；不要覆盖 config 和 data。
```
