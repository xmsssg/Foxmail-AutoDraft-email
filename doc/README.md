# 发货单邮件草稿自动生成工具

## 一、业务需求

发货单邮件是一个高频、重复、但又不能完全自动发送的业务动作。

在实际工作中，业务人员通常需要先制作 Excel 发货单，再打开 Foxmail，手工新建邮件、填写收件人、填写正文、添加附件、保存或发送。这个过程看似简单，但在客户较多、发货频繁、文件目录较深的情况下，很容易出现以下问题：

1. 邮件主题不统一。
2. 收件人或抄送人填写错误。
3. 忘记添加附件，或附件选错。
4. 同一份发货单重复生成邮件。
5. 历史 Excel 被误处理。
6. 业务人员需要在多个客户文件夹和 Foxmail 之间反复切换。

本工具的定位是：

```text
自动生成 Foxmail 草稿，不自动发送邮件。
```

它帮助业务人员把“重复、机械、容易出错”的草稿创建动作自动化，同时保留最后的人工检查和发送环节。

### 1.1 最终业务流程

```text
业务人员制作 Excel 发货单
        ↓
保存到客户指定文件夹
        ↓
工具扫描新 Excel
        ↓
根据客户配置生成邮件草稿
        ↓
自动保存到 Foxmail 草稿箱
        ↓
业务人员在 Foxmail 中检查
        ↓
业务人员手动发送
```

### 1.2 工具负责什么

| 环节 | 是否由工具负责 |
| --- | --- |
| 选择客户 Excel 文件夹 | 是 |
| 支持客户目录下钻扫描子文件夹 | 是 |
| 按起始时间过滤历史 Excel | 是 |
| 根据客户配置生成收件人、抄送人、正文 | 是 |
| 生成带附件 Foxmail 草稿 | 是 |
| 防止同一文件重复生成草稿 | 是 |
| 记录处理状态和日志 | 是 |
| 人工检查邮件内容 | 否 |
| 点击发送邮件 | 否 |

### 1.3 严格边界

本工具不做以下事情：

```text
不自动发送邮件
不连接 SMTP
不连接 POP
不连接 IMAP
不连接网易云企业邮箱服务器
不读取邮箱服务器内容
不删除 Foxmail 邮件
不解析 Excel 业务内容
不替代业务人员审核
```

这条边界非常重要：工具只把邮件准备好，最终是否发送仍由业务人员在 Foxmail 中决定。

## 二、使用方法

### 2.1 推荐安装目录

建议将发布包解压到：

```text
D:\AutoEmail
```

推荐目录结构：

```text
D:\AutoEmail
├─ app
│  ├─ 发货单邮件草稿自动生成工具.exe
│  └─ _internal
├─ config
│  ├─ customers.json
│  └─ settings.json
├─ data
│  └─ draft_records.sqlite
├─ logs
│  └─ app.log
├─ output
│  ├─ generated_eml
│  └─ generated_mapi_xml
├─ doc
└─ 启动发货单草稿工具.vbs
```

首次启动时，程序会自动创建 `config`、`data`、`logs`、`output` 等运行目录。

### 2.2 启动程序

业务人员日常启动入口：

```text
启动发货单草稿工具.vbs
```

也可以直接运行：

```text
app\发货单邮件草稿自动生成工具.exe
```

程序为桌面应用，不会弹出 CMD 黑色窗口。

### 2.3 首次配置

首次使用时，需要在界面中完成三类配置。

#### Foxmail 配置

进入：

```text
系统设置
```

配置：

| 字段 | 说明 |
| --- | --- |
| Foxmail 程序 | 选择 `Foxmail.exe` |
| 邮箱本地目录 | 选择当前邮箱账号在 Foxmail `Storage` 下的目录 |
| 电脑重启后自动启动 | 勾选后，电脑开机自动启动工具 |
| 自动扫描客户文件夹 | 勾选后，程序按间隔自动扫描 |
| 扫描间隔秒数 | 建议 60 秒，最低不建议低于 10 秒 |

#### 客户配置

进入：

```text
客户配置
```

每个客户可以配置：

| 字段 | 说明 |
| --- | --- |
| 客户编号 | 客户唯一编号，建议使用英文或拼音 |
| 客户名称 | 界面中显示的客户名称 |
| Excel 文件夹 | 该客户发货单保存目录 |
| 文件规则 | 例如 `*送货单*.xlsx, *送货单*.xls` |
| 扫描子文件夹 | 勾选后，会扫描客户目录下所有层级 |
| 起始时间 | 只处理修改时间晚于该时间的 Excel |
| 收件人 | 多个邮箱用英文逗号分隔 |
| 抄送人 | 可为空，多个邮箱用英文逗号分隔 |
| 邮件正文 | 可使用 `{subject}` 代表邮件主题 |

#### 起始时间建议

首次上线时，客户文件夹中往往已有大量历史 Excel。为了避免历史文件全部生成草稿，建议：

```text
客户配置 → 选择客户 → 点击“设为当前时间” → 保存客户配置
```

程序只会处理：

```text
Excel 修改时间 > 起始时间
```

### 2.4 日常使用

业务人员每天只需要：

1. 打开 Foxmail。
2. 打开本工具。
3. 制作 Excel 发货单。
4. 将 Excel 保存到客户指定文件夹。
5. 等待自动扫描，或点击 `立即扫描并生成草稿`。
6. 到 Foxmail 草稿箱检查草稿。
7. 确认无误后手动发送。

### 2.5 自动扫描策略

正式使用建议开启自动扫描：

```text
系统设置 → 自动扫描客户文件夹 → 扫描间隔秒数 60
```

工具启动后会按间隔扫描所有客户目录。若业务人员在目录中复制一份历史 Excel 再编辑保存，只要文件修改时间晚于客户配置的起始时间，下一轮自动扫描就会识别并生成草稿。

为避免重复动作，程序会保证：

```text
上一轮扫描未完成时，不启动下一轮扫描。
```

### 2.6 草稿检查清单

发送前必须检查：

| 检查项 | 内容 |
| --- | --- |
| 收件人 | 是否为正确客户邮箱 |
| 抄送人 | 是否需要抄送，抄送人是否正确 |
| 主题 | 是否等于 Excel 文件名去掉扩展名 |
| 正文 | 是否符合客户要求 |
| 附件 | 是否存在 Excel 附件 |
| 附件内容 | 是否为本次发货单 |

### 2.7 防重复规则

工具会根据：

```text
file_path + file_hash
```

判断文件是否已处理。

如果同一文件内容没有变化，重复扫描不会再次生成草稿。

如果同一路径下 Excel 内容发生变化，工具会认为是新版本，允许重新生成草稿。

### 2.8 版本升级方式

新版发布包采用“程序和业务数据分离”的策略。

升级时只替换：

```text
app
doc
```

不要覆盖：

```text
config
data
logs
output
```

这样可以保留业务电脑上已经配置好的客户、Foxmail 路径、处理记录和备查文件。

升级前建议备份：

```text
config
data
```

## 三、技术框架介绍

### 3.1 总体架构

工具采用本地桌面应用架构，核心目标是稳定、透明、可维护。

```text
桌面界面
        ↓
客户配置 / 系统配置
        ↓
文件扫描器
        ↓
文件稳定检测
        ↓
Hash 防重复
        ↓
草稿生成器
        ↓
Foxmail MAPI XML 导入
        ↓
SQLite 状态记录
```

### 3.2 技术选型

| 模块 | 技术 |
| --- | --- |
| 桌面界面 | Python Tkinter |
| 打包方式 | PyInstaller onedir |
| 本地数据库 | SQLite |
| 日志 | Python logging + 滚动日志 |
| 邮件草稿备查 | `.eml` |
| Foxmail 草稿生成 | Foxmail MAPI XML |
| 附件处理 | 本地文件复制到 `%TEMP%\FMMAPI` |
| 防重复 | SHA256 |

### 3.3 Foxmail 导入机制

工具不使用 SMTP，也不直接连接邮箱服务器。

实际导入方式为：

```text
生成 FMMapi.xml
        ↓
复制 Excel 附件到 %TEMP%\FMMAPI
        ↓
调用 Foxmail.exe MAPI: FMMapi.xml
        ↓
Foxmail 打开写信窗口
        ↓
工具执行 Ctrl+S
        ↓
草稿保存到 Foxmail 草稿箱
```

该方式已经验证可以生成带附件草稿，并且重启 Foxmail 后草稿仍存在。

### 3.4 核心模块

| 模块 | 职责 |
| --- | --- |
| `scanner.py` | 扫描客户 Excel 文件夹，支持子文件夹和起始时间过滤 |
| `file_stability.py` | 判断 Excel 是否保存完成 |
| `duplicate_guard.py` | 计算 SHA256，避免重复处理 |
| `eml_builder.py` | 生成 `.eml` 备查文件 |
| `mapi_xml_builder.py` | 生成 Foxmail MAPI XML |
| `foxmail_mapi_importer.py` | 调用 Foxmail 并保存草稿 |
| `draft_generator.py` | 串联扫描、生成、导入、记录流程 |
| `repository.py` | 管理 SQLite 处理记录 |
| `desktop_app.py` | 桌面配置和操作界面 |

### 3.5 数据目录策略

为了支持长期版本迭代，程序目录和业务数据目录分离。

| 目录 | 内容 | 升级策略 |
| --- | --- | --- |
| `app` | 程序本体 | 可覆盖 |
| `config` | 客户和系统配置 | 不覆盖 |
| `data` | SQLite 数据库 | 不覆盖 |
| `logs` | 日志 | 不覆盖 |
| `output` | `.eml`、XML 备查文件 | 不覆盖 |
| `doc` | 文档 | 可覆盖 |

### 3.6 状态设计

| 状态 | 含义 |
| --- | --- |
| `pending` | 已发现，等待处理 |
| `generated` | 草稿描述文件已生成 |
| `importing` | 正在导入 Foxmail |
| `imported` | 已成功保存到 Foxmail 草稿箱 |
| `failed` | 生成或导入失败 |
| `ignored` | 被规则忽略 |

最终成功状态为：

```text
status = imported
import_status = success
```

### 3.7 安全与可控性

工具的设计原则是“自动准备，人工确认”。

它不会越过业务人员的最终审核动作，也不会直接触碰邮箱服务器协议。所有行为都发生在业务人员本机，并以 Foxmail 草稿箱为交付终点。

这使得工具既能提升效率，又不会改变企业邮件发送责任链。
